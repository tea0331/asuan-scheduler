## 数据格式

### 标准数据格式
```json
{
  "lottery": "SSQ",
  "issue": "2025001",
  "date": "2025-01-02",
  "numbers": [1, 2, 3, 4, 5, 6, 7],
  "sales": 0,
  "pool": 0
}
```

### 字段说明
- `lottery`: 彩票类型（"SSQ" 或 "DLT"）
- `issue`: 期号（7 位数字，如 "2025001"）
- `date`: 开奖日期（YYYY-MM-DD 格式）
- `numbers`: 开奖号码数组
  - SSQ: [红1, 红2, 红3, 红4, 红5, 红6, 蓝]
  - DLT: [前1, 前2, 前3, 前4, 前5, 后1, 后2]
- `sales`: 销量（暂无数据）
- `pool`: 奖池（暂无数据）

## 已知限制和改进计划

### 当前限制
1. **日期字段为空**: 
   - 原因: 500.com 页面 HTML 结构复杂，日期和期号在同一行但未正确提取
   - 影响: 无法通过日期查询数据
   - 改进: 后续任务中补充日期提取逻辑

2. **销量和奖池数据为 0**:
   - 原因: 原始数据未包含这些信息
   - 影响: 无法进行销量和奖池分析
   - 改进: 考虑从其他数据源（如官方网站 API）补充

3. **数据更新机制**:
   - 当前: 手动执行采集脚本
   - 改进: 集成到定时任务（如 cron 或 TaskFlow）
