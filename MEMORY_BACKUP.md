## 教训

- `git pull` 在服务器上可能因网络慢卡住，用 `git fetch` + `git reset --hard` 更可靠
- `generate_full_daily.py` 在交互式 terminal 里可能被 SIGKILL，用 `nohup` 后台跑
- 本地修改（generate_full_daily.py / weight-config.json）拉取新代码前要 `git stash` 或 `git checkout --`
- 不要编造诊断结果（曾谎报"磁盘98%满"），不确定就说不确定
