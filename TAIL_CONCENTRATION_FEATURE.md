# 尾数集中度功能集成说明

## 📋 功能概述

基于抖音视频"双色球计算公式"中的**尾数集中度**选号方法，为JinZhu系统（v8.6）新增的特征维度。

## 🎯 核心逻辑

### 尾数集中度评分标准

| 情况 | 示例 | 评分 | 说明 |
|------|------|------|------|
| 1组重复2次 | 03/13 | 1.0 | 理想状态 |
| 1组重复3次 | 03/13/23 | 0.9 | 较好 |
| 2组各重复2次 | 03/13, 07/17 | 1.0 | 理想状态 |
| 2组(3次+2次) | 03/13/23, 07/17 | 0.8 | 可接受 |
| 完全无重复 | 01/02/03/04/05/06 | 0.6 | 过度分散 |
| 3组重复 | 03/13, 07/17, 01/11 | 0.4 | 过多 |
| >=4次重复 | 03/13/23/33 | 0.2~0.3 | 过度集中 |

### 自动调整机制

当5注推荐的平均尾数集中度评分 < 0.6时：
1. 自动检测过度集中的尾数（重复>=3次）
2. 替换1-2个该尾数的号码（保留2个，其余替换为不同尾数）
3. 保持随机性（从候选池随机选号）

## 📝 修改文件清单

### 1. `weight-config.json`
新增参数：
```json
{
  "tail_concentration": 0.10  // 尾数集中度权重（默认10%）
}
```

### 2. `jin_zhu.py`

#### 新增函数：
- `_tail_concentration_score(numbers)` - 计算尾数集中度评分
- `_apply_tail_concentration(recommendations, game)` - 应用尾数集中度调整

#### 修改函数：
- `DEFAULT_MODEL` - 添加`tail_concentration`默认参数
- `PARAM_BOUNDS` - 添加`tail_concentration`边界(0.00, 0.25)
- `_gen_ssq()` - 在返回前调用`_apply_tail_concentration()`
- `_gen_dlt()` - 在返回前调用`_apply_tail_concentration()`

## 🔧 使用方法

### 1. 调整权重

通过`weight-config.json`的`tail_concentration`参数控制影响力度：

```json
{
  "tail_concentration": 0.15  // 提高到15%，加强尾数集中度影响
}
```

**建议范围**：0.05 ~ 0.20

### 2. 禁用功能

设置权重为0即可：

```json
{
  "tail_concentration": 0.00
}
```

### 3. 查看调整日志

启用后，核心注A被调整时会输出日志：

```
[JinZhu] [INFO] [TailConcentration] 调整核心注A: 3重复4次 → 替换2个 → 新评分0.90
```

## ⚠️ 注意事项

1. **不影响其他权重** - 尾数集中度是**独立评分维度**，不影响freq/miss/trend/zone等原有权重
2. **只在必要时调整** - 如果5注推荐的尾数集中度都已经>=0.6，不会强制调整
3. **保持随机性** - 替换号码时从候选池**随机选**，不破坏JinZhu的随机扰动机制
4. **适用于SSQ和DLT** - 双色球和大乐透都支持（PLN/LTN/QXC暂不支持）
