## 三、禁止行为

| ❌ 禁止 | 为什么 |
|---------|--------|
| 修改 `generate_full_daily.py` 的AI prompt | prompt=邪修的灵魂，改了日报就废了 |
| 修改6板块结构/标题/顺序 | 6板块是契约，不能增删改 |
| 创建"简化版"日报生成逻辑 | 历史证明简化=阉割=降级 |
| 在scheduler里硬编码新闻内容 | 调度层不生成内容，这是铁律 |
| 修改降级逻辑(如删掉某个板块的降级) | 降级也必须6板块齐全 |
| 修改邪修引擎(传导链模板/逆潮模式) | 邪修进化由WorkBuddy迭代 |
| 修改质量守护的检查规则 | 规则=底线，不能自己给自己放水 |

---

## 四、日报迭代流程

```
刘老板提需求 → WorkBuddy设计+实现 → 推送到GitHub
    → 阿策 git pull + 执行部署 → 验证日报输出 → 反馈结果
```

**任何日报架构改动，必须走这个流程。阿策不准自作主张改日报代码。**

---

## 五、违规处理

| 违规次数 | 处理 |
|----------|------|
| 第1次 | 警告，回滚改动，由WorkBuddy重新确认 |
| 第2次 | 该任务标记为未完成，日报代码锁定 |
| 第3次 | 暂停服务器git写权限，所有改动必须经WorkBuddy审查后执行 |
