# 阿策铁律 - 第9条：内容契约律

> 生效日期：2026-06-02
> 级别：**强制执行**，违反即视为日报任务未完成
> 触发场景：日报内容被缩减/降级/硬编码/板块缺失

---

## 核心原则

**日报6大板块是不可删减的契约。任何板块缺失、硬编码替代AI生成、内容缩减，均视为任务未完成。**

---

## 一、日报内容契约

### 1.1 六大板块（缺一不可）

| # | 板块名称 | 必须包含 | 最低字数 |
|---|----------|----------|----------|
| 一 | 每日资讯 | 3+小节，每条有💰落地动作 | 200字 |
| 二 | 市场/中间人缺口扫描 | 缺口类型+收钱模式+规避路径+窗口期 | 150字 |
| 三 | 逆潮观察 | 1-3个反直觉信号+逆向下注方向 | 100字 |
| 四 | 深度传导分析 | 5层传导推导+天之道+邪修之道 | 300字 |
| 五 | 避坑提醒 | 1-2个看似机会实际是坑的下注+止损 | 100字 |
| 六 | 今日邪修金句 | AI根据当日新闻生成，1句话 | 20字 |

### 1.2 总体标准

- 日报总字数 ≥ 2000字
- 6板块必须全部出现，标题格式固定
- 每个板块必须有实质内容，不允许空壳标题+占位文字

---

## 二、禁止行为

### 2.1 内容降级禁令

| ❌ 禁止 | ✅ 正确 |
|---------|---------|
| 硬编码固定传导链（每天内容一样） | AI根据当日新闻生成传导链 |
| 硬编码固定邪修金句 | AI根据当日新闻生成金句 |
| 删除某个板块（6个缺1个） | 6个板块必须全部存在 |
| 板块标题+占位文字（"今日暂无"） | 即使降级也写明原因+保留板块结构 |
| 用百度热搜标题替代AI分析 | 热搜仅作素材，AI分析是必须 |
| "AI生成失败，请手动分析" | 修通AI，不能让用户手动 |

### 2.2 降级规范

如果API确实不通（需curl证据），降级模式**也必须6板块齐全**：


## 三、质量守护机制

### 3.1 自动验证

`daily_report_guard.py` 在每次日报发送前自动运行，检查：
1. 6板块是否齐全
2. 每板块字数是否达标
3. 是否有硬编码内容
4. 是否有降级标志
5. 总字数是否 ≥ 2000字

### 3.2 守护集成

在 `scheduler_simple.py` 发送前加入守护检查：

```python
from daily_report_guard import validate_report

result = validate_report(content)
if not result["valid"]:
    logging.error(f"[P0] 日报质量不通过: {result['errors']}")
    # 不发送，等待修复
else:
    send_email(subject, content)
```

### 3.3 日报对比检查

每日自动对比前一天的日报：
- 邪修金句是否相同 → 硬编码检测
- 传导链是否完全相同 → 硬编码检测
- 总字数差异 > 30% → 可能被缩减

## 四、架构约束

### 4.1 单轨制

- **scheduler_simple.py**：只负责调度+发送，不生成新闻内容
- **generate_full_daily.py**：唯一内容生成器，包含完整6板块prompt
- **daily_report_guard.py**：质量守护，发送前验证

### 4.2 禁止双轨

- ❌ 不许在scheduler_simple.py里硬编码新闻内容
- ❌ 不许创建"simple版"新闻生成逻辑
- ✅ scheduler只能调generate_full_daily的函数

### 4.3 内容与调度分离

```
scheduler_simple.py（调度层）
  ├── generate_news_section() → generate_full_daily.py
  ├── generate_lottery_section() → generate_full_daily.py → jin_zhu.py
  ├── generate_taiwan_section() → generate_taiwan.py
  └── daily_report_guard.py（验证层，发送前检查）
```

## 五、违规处理

| 违规次数 | 处理 |
|----------|------|
| 第1次 | 警告，要求补充缺失板块 |
| 第2次 | 日报标记为未完成，重新生成 |
| 第3次 | 禁止独立修改日报相关代码，需阿算审查 |

| 日期 | 降级行为 | 后果 |
